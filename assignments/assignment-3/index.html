Assignment 3 - UX Flow Chart

FilmFreeway - UX

Note

5/6 scenarios start with the user on the homepage and establishing their account status before being taken to the "Browse Festivals page". I did this because in order to submit to FilmFreeway/handle your projects you need an account and I decided to not go with the prefix that the user is already logged in, or that they start the login/sign up process at a later stage in the scenario.

Bigger versions of the flow charts can be found in the "big charts" tab or you can right click on the images to open them in a new tab

The Legend

The legend features in all flow charts. Doing this allows the viewer to easily check what a symbol in the chart means.

  1. The first symbol is a red oval that indicates the start and end points of the chart
  2. The second symbol is a black arrow showing the flow of the chart
  3. The third symbol is a green rectangle which indicates processes/actions carried out by the user
  4. The fourth symbol is a yellow diamond that denotes when a decision must be made in the flow/a condition is being checked
  5. The last symbol is a blue rectangle with the top two corners rounded, this is used to show a web page the user lands on when going through the flow.

Scenario 1:

The user tries to submit a film to a festival without having a film in their FilmFreeway (FF) account.

The user starts on the FF Homepage and either logs in or signs up which we can see represented in a decision diamond, then they are taken to the "Browse Festivals" page. In ths scenario the users presses the submit now button for the festvial they wish to submit to but the user has no projects set up in their FilmFreeway account. As such a pop up asks them to add a project, which is another decision diamond in the chart. If they select no, the user is returned to the "Browse Festivals" page, if they select yes they then go through the process of adding a project.

They must upload their project info and their video file. Then the user can return to the "Browse Festivals" page where they can now submit a film. To do this the user starts again by pressing the submit button, they then choose a category in this festival and add their choice to the cart/checkout. After this they must input their card details to then pay/complete their order.

Scenario 2:

The user searches for festivals that accept music videos and succesfully submits their project to a film festival which asks them to make a fee payment.

The user starts on the FF Homepage and either logs in or signs up which we can see represented in a decision diamond, then they are taken to the "Browse Festivals" page. From here the user searchs for music video awards in the search bar and when they find one they press the submit button.

They then choose a category in this festival and add their choice to the cart/checkout. After this they must input their card details to then pay/complete their order which will bring them to the end of this scenario.

Scenario 3:

The user changes the poster image of one of their projects.

The user starts on the FF Homepage and either logs in or signs up which we can see represented in a decision diamond, then they are taken to the "Browse Festivals" page. The user selects the my projects tab to then view their project.

Here they can press the edit button that will allow them to pick the option to change the poster image. They can select an image from the file explorer that opens on their screen, this completes this scenario as there is no further step required to save the change.

Scenario 4:

The user looks for festivals that don't require an application fee.

This is the one scenario in which I didn't opt to start with the user having to log in/sign in because the actions in this scenario don't require an account.

Instead the user lands on the FF homepage and selects the "Browse Festivals" page. To look for a festival that doesn't require an application fee the user sets the entry fee slider to 0.

Scenario 5:

The user withdraws from a festival submission.

The user starts on the FF Homepage and either logs in or signs up which we can see represented in a decision diamond, then they are taken to the "Browse Festivals" page. From here they select the submissions tab and click on the drop down menu.

The user selects withdraw from festival for their relevant film, which prompts a pop up, represented as a decision diamond. If the user confirms their withdrawal their submission is withdrawn and the user is returned to the submissions page. If the user opts not to withdraw their film, their film remains submitted and they are returned to the submission page.

(Bonus) Scenario 6:

The user deletes their FF account

The user starts on the FF Homepage and either logs in or signs up which we can see represented in a decision diamond, then they are taken to the "Browse Festivals" page.

The user selects the dashboard and then account settings. In here the user selects delete account, FF requires the user to press it a second time before a pop up checks for confirmation of the deletion (show as a decision diamond). If the user selects yes, their account is deleted, if they select no their account isn't deleted they are returned to the account settings.

References:

Nunes, Jason, and Andy Pratt. Interactive Design: An Introduction to the Theory and Application of User-Centered Design. 1st Ed., Rockport Publishers, 2012.
